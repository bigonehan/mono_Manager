infer_draft_item prompt
- 반드시 언급된 코드/문서 형식을 정확히 지킨다.
- 출력은 YAML만 반환한다.
- 템플릿의 설명용 예시값/주석/placeholder는 최종 출력물에 포함하지 않는다.
- 최상위 키는 `type`, `domain`, `depends_on`, `rule`, `step`, `constraints`만 사용한다.
- 각 키 값은 문자열 리스트 또는 문자열(type)로 작성한다.
- `project.md`, `plan_yaml`, `name`, `input_rules`, `input_steps`를 함께 읽고 해당 draft_item의 책임을 추론한다.
- domain은 project.md의 domains 중 적합한 항목을 선택한다.


project_md:
# info
name : tmux_split_verify
description : tmux split verify
spec : tmux, bash, shell scripting
path : /home/tree/project/rust-orc/.tmp_tmux_iso

# features
- workspace_bootstrap

# rules
- 모든 자동화는 `tmux` 세션/윈도우/패인 존재 여부를 선검증한 뒤 실행한다.
- 쉘 스크립트는 `bash` 기준으로 작성하고, 실행 옵션은 `set -euo pipefail`을 기본으로 유지한다.
- 명령 실행 결과는 성공/실패 기준과 종료 코드를 명확히 기록한다.
- 경로 기준은 프로젝트 루트(`/home/tree/project/rust-orc/.tmp_tmux_iso`)를 우선 사용한다.
- 검증 로직은 재실행 가능해야 하며 동일 입력에서 동일 결과를 보장한다.

# constraints
- 지원 범위는 `tmux`, `bash`, 표준 shell scripting으로 제한한다.
- 대화형 입력 의존 없이 비대화형 실행이 가능해야 한다.
- 외부 네트워크/외부 서비스 의존 검증을 추가하지 않는다.
- 프로젝트 외부 디렉터리의 파일 생성/수정은 금지한다.
- 실패 시 즉시 중단하고 원인 파악에 필요한 최소 로그를 남겨야 한다.

# domains
## workspace_bootstrap
### states
- not_initialized
- session_checked
- workspace_ready
- split_configured
- verification_passed
- verification_failed
### action
- 작업 디렉터리 유효성 확인
- `tmux` 세션 존재 여부 확인 및 생성
- 대상 윈도우/패인 분할 구성 적용
- 분할 결과(패인 수/레이아웃) 검증
- 검증 결과 로그 출력 및 종료 코드 반환
### rules
- 상태 전이는 순차적으로 진행하며 단계 건너뛰기를 허용하지 않는다.
- `verification_failed` 상태에서는 후속 분할 작업을 수행하지 않는다.
- 이미 구성된 세션이 있으면 재사용하고 중복 생성하지 않는다.
- 검증 기준은 패인 개수와 레이아웃 일치 여부를 모두 포함한다.
### constraints
- `tmux` 명령 실패 시 재시도는 1회 이내로 제한한다.
- 검증 대상 세션/윈도우 이름은 스크립트 입력값 또는 고정 규칙 중 하나로 일관되게 처리한다.
- 상태 판별은 `tmux list-sessions`, `tmux list-windows`, `tmux list-panes` 결과만 사용한다.
