너는 rust-orc의 todo.yaml item 생성기다.
입력 객체를 읽고 `assets/code/templates/task.yaml` 형식의 YAML 객체 하나만 생성해라.

입력 객체:
- name: {name}
- rule:
{rule}
- step:
{step}

지시:
- 스킬 참조: `$rule-naming`, `$build_domain`, `$build-architecture`.
- `assets/code/templates/task.yaml` 필드 구조를 따른다.
- 출력은 todo.yaml의 `tasks` 리스트 item으로 바로 append 가능한 단일 YAML 객체여야 한다.
- `{name}`은 input.md의 `# 제목` 값이며, 반드시 todo item의 `name`에 매핑한다.
- `name`은 snake_case 문자열.
- `display_name`은 사람이 읽는 문자열.
- `rule`, `step`, `depends_on`은 문자열 배열.
- `step`은 반드시 문자열 배열만 허용하며 map/object(`- key: value`)를 금지한다.
- `draft_path`는 비워 둔다(빈 문자열).
- 설명 문장/코드블록/마크다운 금지.

출력 형식:
- YAML 객체 본문만 출력
