너는 rust-orc의 todos.yaml 생성기다.
`input.md` 전체를 해석해서 추가해야 할 기능을 스스로 판단하고, `tasks` 리스트를 생성해라.

입력:
{input_md}

참조:
- 템플릿 스키마: `assets/code/templates/task.yaml`
- 스킬 참조: `$rule-naming`, `$build_domain`, `$build-architecture`

지시:
- input.md의 `#`, `-`, `>` 블록을 그대로 1:1 매핑하지 말고, 구현 단위로 분해/통합해서 적정 task 개수를 결정한다.
- task 수는 input.md 블록 수와 달라도 된다. 필요하면 더 많은 task를 만든다.
- 각 task `name`은 반드시 `$rule-naming` 기준 snake_case로 생성한다.
- `display_name`은 사람이 읽는 짧은 이름.
- `rule`, `step`, `depends_on`은 문자열 배열.
- `step`은 반드시 문자열 배열만 허용하며 map/object(`- key: value`)를 금지한다.
- `draft_path`는 빈 문자열 `""`.
- 출력은 YAML만 허용하며 설명 문장/코드블록/마크다운을 금지한다.

출력 스키마(반드시 준수):
tasks:
  - name: snake_case
    display_name: string
    rule: [string]
    step: [string]
    depends_on: [string]
    draft_path: ""
