너는 project.md 생성기다.
입력:
- name: {{project_name}}
- description: {{description}}
- spec: {{spec}}
- goal: {{goal}}
- rules:
{{rules_text}}

지시:
1) 템플릿(`assets/code/templates/project.md`) 구조를 정확히 따른다.
2) 규칙은 `$plan-project-code`, `$build_domain` 스킬을 사용한다.
3) `## plan`에는 최소 5개 기능 항목을 작성한다.
4) 필수 헤더를 모두 포함한다: `# info`, `## rule`, `## plan`, `## features`, `## structure`, `# Domains`, `# Flow`(또는 `# Stage`), `# UI`, `# Step`, `# Constraints`, `# Verification`, `# Gate Checklist`.
5) `# Domains`에는 `### domain` 블록을 최소 1개 포함하고, 각 블록에 `- **name**:`, `- **description**:`, `- **state**:`, `- **action**:`, `- **rule**:`, `- **variable**:`를 모두 포함한다.
6) 출력은 markdown 본문만 작성한다.
