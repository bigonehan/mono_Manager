# AGENTS

## UI Color Rule
- 앞으로 UI 관련 작업을 할 때 색상값은 `configs/style.yaml`을 참조한다.

## YAML Handling Rule
- 앞으로 YAML 관련 파일 작업 시, 문서/설명/예시는 YAML 실제 값으로 해석되지 않도록 기본적으로 주석 처리한다.
- 실제 파싱에 사용되는 YAML 파일에는 실행에 필요한 순수 데이터만 둔다.
- 템플릿 설명이 필요한 경우, 문서용 템플릿 파일과 실행용 YAML 파일을 분리한다.

## Function Naming Rule
- `flow_` 접두사는 프로그램의 단계 전환/오케스트레이션 흐름 함수에만 사용한다.
- `flow_`는 분기 제어, 단계 진입/종료, 실행 순서 조합처럼 "흐름을 조율"하는 함수에만 허용한다.
- 계산/수정/생성/파싱/로드/추출/변환/전송 같은 직접 동작 함수에는 `flow_`를 사용하지 않는다.
- 단순 데이터 추가/전송/실행 같은 직접 동작 함수는 역할 기반 접두사를 사용한다.
- 메시지 전송/추가 성격 함수는 `send_` 접두사를 우선 사용한다.
- 런타임 메인 오케스트레이션 진입점(`flow_main`)은 예외적으로 유지할 수 있다.
