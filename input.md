# rust_cli_workspace
- Rust CLI 워크스페이스는 명령 단위로 모듈을 분리하고 단일 책임 원칙을 유지한다.
- 실패 상태에서 성공 메시지를 출력하지 않는 실행 상태 체계를 유지한다.
> 워크스페이스 기본 구조와 공통 실행 엔트리를 구성한다.
> 파싱·실행·결과 보고 상태 전이를 표준 흐름으로 연결한다.

# project_documentation
- 프로젝트 문서 기준 파일은 반드시 `./.project/project.md` 단일 경로를 사용한다.
- 규칙·제약·도메인 항목은 모두 마크다운 불릿 리스트 형식을 유지한다.
> 문서 로더가 기준 경로만 읽도록 고정한다.
> 도메인 섹션 구조와 불릿 규칙 검증을 추가한다.

# cli_rust_orchestra
- CLI 오케스트레이션은 정의된 feature 명령만 디스패치한다.
- 명령 실행 전 입력 유효성을 확인하고 상태를 `cli_parsing`에서 `cli_executing`으로 전이한다.
> 루트 명령에서 서브커맨드 디스패처를 구성한다.
> 실행 결과를 성공·실패 코드와 메시지로 일관되게 반환한다.

# cli_help
- `help` 명령은 문서화된 명령만 노출한다.
- 정의되지 않은 명령 별칭은 허용하지 않는다.
> 사용 가능한 명령 목록과 인자를 출력한다.
> 잘못된 입력 시 도움말과 오류 코드를 함께 반환한다.

# cli_create_code_draft
- 코드 초안 생성은 입력 유효성 검사 후 실행한다.
- 문서화된 목적 외 동작을 추가하지 않는다.
> 초안 생성 입력을 파싱하고 필수 필드를 검증한다.
> 도메인·규칙 기반 초안 파일을 생성하고 결과를 보고한다.

# cli_impl_code_draft
- 초안 구현 명령은 초안 상태 검증 후 실행한다.
- 실패 시 부분 성공 메시지를 출력하지 않는다.
> 구현 대상 초안을 로드하고 적용 가능 여부를 점검한다.
> 구현 작업을 수행하고 변경 결과를 상태별로 보고한다.

# cli_test
- `test` 명령은 문서화된 테스트 실행 범위만 수행한다.
- 테스트 실패는 즉시 `cli_failure`로 전환해 보고한다.
> 프로젝트 테스트 실행기를 호출한다.
> 통과·실패 결과와 요약을 표준 출력 형식으로 반환한다.

# cli_check_task
- 작업 점검 명령은 입력 태스크의 형식과 필수 항목을 검증한다.
- 검증 규칙은 문서 기준과 일치해야 한다.
> 태스크 입력을 파싱하고 규칙 위반 항목을 수집한다.
> 점검 결과를 오류 목록 또는 통과 상태로 출력한다.

# cli_check_draft
- 초안 점검 명령은 draft 구조와 필수 섹션 존재를 확인한다.
- 규칙 위반 시 자동 수정 없이 실패를 명확히 반환한다.
> 대상 초안을 로드하고 형식 규칙을 검사한다.
> 점검 리포트를 생성해 성공·실패 상태를 출력한다.

# cli_open_ui
- UI 열기 명령은 오케스트레이션 상태 흐름을 유지한다.
- 접근성 기본 속성과 반응형 정보 구조를 훼손하지 않는 실행 경로만 허용한다.
> UI 실행 트리거를 파싱해 핸들러를 호출한다.
> UI 프로세스 상태를 추적하고 결과를 CLI에 보고한다.

# cli_init_code_project
- 프로젝트 초기화는 `./.project/project.md` 기준 체계를 전제로 동작한다.
- 초기화 시 문서화되지 않은 추가 자산 생성은 금지한다.
> 코드 프로젝트 기본 디렉터리와 필수 파일을 생성한다.
> 초기화 완료 상태를 검증하고 결과를 반환한다.

# cli_init_code_plan
- 계획 초기화는 `plan.yaml` 구조 규칙을 준수한다.
- `planned/worked/complete` 상태 구분을 유지한다.
> 계획 파일의 기본 스키마를 생성한다.
> 목표·도메인·draft 섹션 유효성을 점검해 저장한다.

# cli_add_code_plan
- 계획 추가 명령은 기존 `plan.yaml` 구조를 깨지 않고 갱신한다.
- 중복 항목 추가를 허용하지 않는다.
> 추가 대상 항목을 파싱하고 중복 여부를 검사한다.
> `planned` 목록에 반영하고 변경 결과를 보고한다.

# cli_add_code_draft
- 초안 추가 명령은 계획 항목과의 정합성을 확인한다.
- 문서에 없는 목적의 초안 유형은 허용하지 않는다.
> 신규 초안 메타데이터를 검증한다.
> 초안 저장 후 인덱스와 상태를 갱신한다.

# cli_add_code_draft_item
- 초안 아이템 추가는 대상 초안 존재 확인 후 수행한다.
- 아이템 형식은 마크다운 불릿 규칙을 유지한다.
> 초안 파일을 로드하고 추가 위치를 계산한다.
> 아이템을 삽입하고 무결성 점검 후 저장한다.

# cli_check_code_draft
- 코드 초안 점검은 도메인·rules·constraints 반영 여부를 확인한다.
- placeholder, 주석성 예시값, 설명용 텍스트 포함을 허용하지 않는다.
> 초안 본문을 스캔해 형식/규칙 위반을 탐지한다.
> 위반 목록을 리포트하고 상태를 반환한다.

# cli_auto
- 자동 실행 명령은 문서화된 순서와 범위 내에서만 동작한다.
- 단계 실패 시 후속 단계를 무조건 진행하지 않는다.
> 자동 파이프라인 단계들을 순차 디스패치한다.
> 각 단계 결과를 집계해 최종 상태를 보고한다.

# cli_send_tmux
- tmux 전송 명령은 명시된 세션·패널 대상만 허용한다.
- 전송 실패를 성공으로 표시하지 않는다.
> 대상 세션과 패널 식별자를 검증한다.
> 명령 문자열을 전송하고 실행 결과를 반환한다.

# cli_enter
- enter 명령은 지정된 실행 컨텍스트 진입만 수행한다.
- 정의되지 않은 진입 별칭은 허용하지 않는다.
> 진입 대상과 옵션을 파싱한다.
> 컨텍스트 전환 후 상태를 점검해 보고한다.

# cli_chat
- chat 명령은 오케스트레이션 규칙에 맞는 대화 실행만 처리한다.
- 문서 기반 입력 검증 없이 실행하지 않는다.
> 채팅 입력을 검증하고 실행 컨텍스트를 준비한다.
> 응답 처리 후 결과 상태를 CLI로 반환한다.

# cli_chat_wait
- chat_wait 명령은 대기 상태와 완료 상태를 명확히 구분한다.
- 타임아웃·실패 상태를 누락 없이 보고한다.
> 대기 루프를 시작하고 이벤트 완료를 감시한다.
> 완료·타임아웃·오류 결과를 상태 코드와 함께 출력한다.
